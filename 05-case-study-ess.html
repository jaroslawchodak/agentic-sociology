5  Case Study — European Social Survey

Ten rozdział to najważniejsza część książki. Wszystko, co dotąd opisałem, ma sens dopiero wtedy, gdy zobaczysz pełny pipeline badawczy wykonany przez agenta — od pustego folderu do publikowalnego raportu.

5.1 Pytanie badawcze

Czy osoby o wyższym zaufaniu społecznym, lepszym zdrowiu, wyższym dochodzie i wyższym poziomie edukacji deklarują wyższe zadowolenie z życia?

Dane: European Social Survey (publiczne, dostępne na europeansocialsurvey.org).

Model:

stflife ~ ppltrst + health + hinctnta + eduyrs + agea + gndr + cntry

5.2 Folder demo — punkt wyjścia

ess-demo/
  AGENTS.md            ← konstytucja agenta
  CONTEXT.md           ← opis projektu i pytanie
  WORKFLOW.md          ← procedura analizy
  prompt-wykonawczy.md ← zlecenie
  data_raw/            ← surowe dane ESS
  data_clean/          ← po czyszczeniu
  scripts/             ← skrypty pomocnicze
  outputs/             ← raport + wykres
  agent_logs/          ← ślad decyzji

Trzy pliki .md mówią agentowi, kim jest, co bada i jak ma to zrobić.

Cały folder znajdziesz w repozytorium pod ścieżką ESS-demo/ — możesz go skopiować i uruchomić u siebie.

5.3 Pipeline wykonany przez agenta

Wszystkie te kroki agent wykonuje sam, na podstawie procedury zapisanej w WORKFLOW.md. Twoja rola: napisać prompt wykonawczy i sprawdzić wynik.

5.4 Prompt wykonawczy — czyli „zadanie na dziś”

Wykonaj demonstracyjną analizę ESS zgodnie z
AGENTS.md, CONTEXT.md i WORKFLOW.md.

Pytanie badawcze:
Czy osoby o wyższym zaufaniu społecznym, lepszym zdrowiu,
wyższym dochodzie i wyższym poziomie edukacji deklarują
wyższe zadowolenie z życia?

Model:
stflife = ppltrst + health + hinctnta + eduyrs + agea + gndr + cntry

Wykonaj: sprawdzenie danych, czyszczenie, opis,
korelację, wykres, regresję, ostrożną interpretację,
listę ograniczeń.

Zapisz: outputs/raport-ess-stflife.md
        agent_logs/decyzje-ess-stflife.md

To wszystko po polsku. Żadnego kodu napisanego przez prowadzącego.

5.5 Krok 1: Agent pisze skrypt czyszczący

Nie musisz tego napisać

Poniższy skrypt jest przykładem, jak agent sam zaprojektował kod na podstawie procedury z WORKFLOW.md. Pokazuję go tu, żeby zilustrować, co konkretnie dzieje się „pod maską”. Twoim zadaniem jest wyłącznie ocena wyniku.

import pandas as pd

df = pd.read_csv("data_raw/ESS_subset.csv")

cols = ["stflife", "ppltrst", "health", "hinctnta",
        "eduyrs", "agea", "gndr", "cntry"]
df = df[cols]

# Kody braków ESS: 7, 8, 9, 77, 88, 99
missing_codes = {
    "stflife": [77, 88, 99],
    "ppltrst": [77, 88, 99],
    "health":  [7, 8, 9],
    "hinctnta":[77, 88, 99],
    "eduyrs":  [77, 88, 99],
}
for col, codes in missing_codes.items():
    df.loc[df[col].isin(codes), col] = pd.NA

df_clean = df.dropna()
df_clean.to_csv("data_clean/ess_clean.csv", index=False)

5.6 Krok 2: Agent liczy regresję OLS

Adnotacja

Nie musisz znać statsmodels — chodzi tylko o to, że agent sam dobiera bibliotekę odpowiednią do zadania.

import statsmodels.formula.api as smf

model = smf.ols(
    "stflife ~ ppltrst + health + hinctnta + eduyrs "
    "+ agea + C(gndr) + C(cntry)",
    data=df_clean,
).fit()

print(model.summary())

5.9 Co jest socjologicznie ważne w tym pokazie

  • Agent nie pisze interpretacji za nas — proponuje, a my decydujemy.
  • Agent sam zaznacza ograniczenia: wagi ESS, skale porządkowe, cntry jako proste fixed effects zamiast modelu wielopoziomowego.
  • Agent NIE formułuje wniosków przyczynowych z danych przekrojowych.
  • Audit log = polisa ubezpieczeniowa dla recenzentów.
Klucz dydaktyczny

Agent zna metodologię, bo my mu ją wpisaliśmy w AGENTS.md.

Bez pliku kontekstowego agent byłby tylko „mądrzejszym czatem”. Z plikiem — staje się asystentem, który respektuje Twoje zasady metodologiczne. To Ty ustawiasz reguły gry.
